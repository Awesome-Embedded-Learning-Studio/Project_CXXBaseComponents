# 06 - 完整示例：从构建到运行的全流程

## 把所有东西组合起来

前面我们花了五篇文章的篇幅，把 ArgParser 的各个组件逐个拆解开来讲解。现在到了把它们拼到一起的时候了，我们用一个完整的示例程序来演示这些组件是如何协同工作的。

这个示例程序会在源代码目录下的 `src/ArgParser/demo.cpp`，你可以把它编译出来跟着我们一起测试。

## 示例程序的设计

我们的 demo 程序模拟一个简单的文件处理工具。它需要一个输入文件路径（位置参数，必填），一个输出文件路径（命名参数，必填），还有一些可选的控制选项：处理数量、详细模式开关、强制覆盖开关。

```cpp
int main(int argc, char *argv[]) {
    cc::ArgParser parser;

    // 位置参数（必填）
    parser.add_positional("input", "输入文件路径", true);

    // 命名参数
    parser.add_argument("-o", "--output", "输出文件路径", true);
    parser.add_argument("-n", "--count", "处理数量（默认 1）", false);

    // 布尔开关
    parser.add_flag("-v", "--verbose", "启用详细输出模式");
    parser.add_flag("-f", "--force", "强制覆盖已存在文件");
```

首先创建一个 ArgParser 实例，然后依次注册所有需要的参数。注意这里的参数定义顺序——我们先把核心的输入输出参数定义好，然后再定义那些控制行为的可选参数。这种排列方式会让帮助信息看起来更有逻辑层次。

## 解析与错误处理

参数注册完成之后，接下来就是解析命令行输入：

```cpp
    try {
        parser.parse(argc, argv);
    } catch (const std::exception &e) {
        std::println(stderr, "[错误] {}", e.what());
        std::println(stderr, "使用 --help 查看帮助信息");
        return 1;
    }
```

我们把 `parse()` 调用包裹在 try-catch 块里。如果解析过程中出现任何问题——无论是未知参数、缺少必填值，还是格式错误——都会抛出异常，我们在这里捕获并打印友好的错误信息，然后提示用户可以用 `--help` 查看完整的帮助。

## 读取参数值

解析成功之后，我们就可以读取各个参数的值并做相应的处理了：

```cpp
    std::println("=== ArgParser 演示 ===");
    std::println("");

    // 位置参数
    try {
        const std::string input = parser.get<std::string>("input");
        std::println("输入文件:   {}", input);
    } catch (const std::exception &e) {
        std::println(stderr, "读取 input 参数失败: {}", e.what());
    }
```

对于位置参数，我们用 `get<std::string>("input")` 来获取它的值。因为 "input" 是必填参数，正常情况下这里一定能拿到值，但我们还是把它放在 try-catch 里以防万一。

```cpp
    // 命名参数
    try {
        const std::string output = parser.get<std::string>("output");
        std::println("输出文件:   {}", output);
    } catch (const std::exception &e) {
        std::println("输出文件:   (未指定)");
    }
```

对于可选参数，我们需要处理它没有被指定的情况。这里如果 `get()` 抛出异常，我们就打印一个 "(未指定)" 的占位符，而不是让程序崩溃。

```cpp
    // 可选的数值参数
    try {
        const int count = parser.get<int>("count");
        std::println("处理数量:   {}", count);
    } catch (const std::exception &e) {
        std::println("处理数量:   (使用默认值 1)");
    }
```

数值参数的处理方式类似。如果没有指定 count，我们就假设使用默认值 1。在实际的程序中，你可能会在这里用一个变量来存储默认值，或者直接在注册参数时设置默认值（这是 ArgParser 可以继续扩展的一个功能）。

## 检查布尔开关

布尔开关的处理最简单，因为 `get_flag()` 不会抛出异常，它只会返回 true 或 false：

```cpp
    std::println("详细模式:   {}", parser.get_flag("verbose") ? "开启" : "关闭");
    std::println("强制覆盖:   {}", parser.get_flag("force") ? "开启" : "关闭");
```

这里用了三元运算符来把布尔值转换成中文的"开启"或"关闭"，让输出更易读。当然你也可以直接打印 true 或 false，看你的需求。

## 编译与运行

现在让我们把这个程序编译出来并测试一下。在源代码目录下执行：

```bash
g++ demo.cpp ArgParser.cpp -o demo -std=c++23
```

编译成功之后，我们就可以开始测试了。

### 测试场景一：查看帮助

```bash
./demo --help
```

输出应该是这样的：

```
Usage: ./demo <input> [options]

Positional arguments:
  <input>                输入文件路径

Options:
  -o, --output <value>   输出文件路径 [required]
  -n, --count <value>    处理数量（默认 1）
  -v, --verbose          启用详细输出模式
  -f, --force            强制覆盖已存在文件
  -h, --help             Show this help message
```

### 测试场景二：正常调用

```bash
./demo input.txt -o output.txt -n 10 -v
```

输出：

```
=== ArgParser 演示 ===

输入文件:   input.txt
输出文件:   output.txt
处理数量:   10
详细模式:   开启
强制覆盖:   关闭

=== 解析完成 ===
```

### 测试场景三：使用等号格式

```bash
./demo input.txt --output=output.txt --count=5 --force
```

输出：

```
=== ArgParser 演示 ===

输入文件:   input.txt
输出文件:   output.txt
处理数量:   5
详细模式:   关闭
强制覆盖:   开启

=== 解析完成 ===
```

你会发现 `--output=output.txt` 这种等号格式也能正确解析，而且和空格格式是等价的。

### 测试场景四：缺少必填参数

```bash
./demo input.txt
```

输出：

```
[错误] Required args: output
使用 --help 查看帮助信息
```

程序正确地检测到了必填参数缺失，并给出了清晰的错误提示。

### 测试场景五：未知参数

```bash
./demo input.txt -o output.txt --foo
```

输出：

```
[错误] Unknown argument: --foo
使用 --help 查看帮助信息
```

未知参数也会被正确捕获，用户能立刻知道自己输错了什么。

## 调试技巧

在开发过程中，如果你发现参数解析不符合预期，可以在 main 函数里临时加一些调试输出：

```cpp
// 在 parse() 之后添加
std::cout << "解析到的参数数量: " << parser.args_.size() << "\n";
for (const auto& [key, arg] : parser.args_) {
    std::cout << key << ": is_set=" << arg.is_set << ", value=" << arg.value << "\n";
}
```

这样你就能看到每个参数的解析状态，快速定位问题所在。当然，这在正式发布前要记得删掉。

## 下一步可以做什么

这个 demo 程序虽然简单，但它展示了 ArgParser 的完整使用流程。在实际的项目中，你可能需要更复杂的功能，比如默认值支持、多值参数、子命令系统等等。这些都是可以在现有架构基础上扩展出来的。

如果你对 ArgParser 的内部实现感兴趣，可以试着为它添加一个新功能，比如支持默认值。这会让你对 C++ 的模板编程、API 设计有更深的理解。当你把这个东西完全吃透之后，再去阅读那些成熟的命令行解析库的源码，你会发现很多设计思路都是相通的。
